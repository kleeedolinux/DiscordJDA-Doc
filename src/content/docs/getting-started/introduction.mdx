DiscordRDA (Ruby Development API) is a high-performance Ruby library designed from the ground up to address a critical gap in the Ruby ecosystem: the lack of a modern, production-ready Discord library with full support for Discord's latest features.

## Why DiscordRDA Was Created

The Ruby ecosystem has long lacked a comprehensive Discord library that keeps pace with Discord's rapidly evolving API. While other languages like JavaScript and Python have multiple mature options, Ruby developers were forced to either:

- Use outdated libraries that don't support modern Discord features like Slash Commands and Component V2
- Build custom solutions that require significant time and maintenance overhead
- Switch to other languages, abandoning Ruby's elegant syntax and powerful ecosystem

DiscordRDA was built to solve these problems by providing a first-class Discord development experience for Ruby developers, with complete support for modern Discord features including Slash Commands, Context Menus, Modals, Autocomplete, and the latest Component V2 architecture.

## Core Capabilities

DiscordRDA provides everything you need to build sophisticated Discord bots in Ruby:

**Modern Async Architecture**
Built on Ruby 3.0's Fiber scheduler for true concurrency, allowing your bot to handle thousands of connections efficiently without blocking operations or complex thread management.

**Full Discord API Coverage**
Complete implementation of Discord's REST API and Gateway, including all modern interaction types. Whether you need simple message commands or complex multi-step workflows with modals and buttons, DiscordRDA handles it natively.

**Enterprise-Grade Scalability**
Built-in support for sharding, caching layers, and rate limit management. The library handles the complexity of Discord's rate limits automatically, queuing requests intelligently to prevent hitting limits while maximizing throughput.

**Clean, Ruby-Idiomatic Design**
The library embraces Ruby's philosophy of developer happiness with intuitive APIs, sensible defaults, and comprehensive documentation. Entity creation follows factory patterns for consistency, and the event-driven architecture makes bot logic straightforward to implement and maintain.

## Feature Highlights

**Slash Commands and Interactions**
Create, deploy, and manage Discord Slash Commands with full support for subcommands, options, and permissions. Handle all interaction types including buttons, select menus, and modal forms through a unified event system.

**Component V2 Architecture**
Native support for Discord's latest component system, including containers, text displays, media galleries, and file attachments. Build rich, interactive interfaces that go far beyond simple message responses.

**Advanced Event Handling**
Comprehensive event system covering all Discord Gateway events, from basic message creation to voice state updates and guild member changes. Event handlers are easy to register and provide full context for each event type.

**Flexible Caching**
Pluggable caching system with both memory and Redis backends. Cache entities selectively or comprehensively based on your bot's needs, with automatic cache invalidation and pattern-based expiration.

**Development and Deployment Tools**
Hot reload capabilities for rapid development, zero-downtime shard rebalancing for production scaling, and session transfer support for seamless bot updates without disconnecting users.

## Quick Start Example

```ruby
require 'discord_rda'

bot = DiscordRDA::Bot.new(
  token: ENV['DISCORD_TOKEN'],
  intents: [:guilds, :guild_messages, :message_content]
)

bot.on(:message_create) do |event|
  if event.content == '!ping'
    event.message.respond(content: 'Pong!')
  end
end

bot.run
```

This example demonstrates the simplicity of DiscordRDA's API while hinting at the power underneath. The same intuitive pattern extends to complex features like Slash Commands with multiple options, modal interactions with validation, and event handling across multiple shards.

## System Requirements

DiscordRDA requires Ruby 3.0 or higher to leverage the Fiber scheduler for optimal performance. The library is tested and supported on Linux, macOS, and Windows platforms.

For production deployments, we recommend Ruby 3.2+ for the best balance of performance and compatibility.
